## Description

Xây dựng hạ tầng xử lý lỗi toàn cục cho app:

1. **AppException types** — phân loại exception rõ ràng tại `core/error/`
2. **Dio interceptor** — retry tự động 3 lần với exponential backoff khi gặp lỗi network/server
3. **Error snackbar provider** — hiển thị lỗi thân thiện lên UI từ bất kỳ layer nào

Mục tiêu:
- Tất cả lỗi đều được type-safe (không dùng `Exception` generic)
- Retry logic tập trung tại Dio interceptor, các layer trên không cần biết
- UI chỉ cần `ref.watch(errorSnackbarProvider)` để nhận và hiển thị lỗi

## Notes

- `AppException` dùng `sealed class` để exhaustive matching tại compile time
- `AppFailure` là mirror của `AppException` nhưng ở domain layer — tách biệt để domain không phụ thuộc Dio
- `dioClientProvider` là provider duy nhất cho Dio instance — các datasource phải inject qua provider này, không tạo Dio riêng
- Exponential backoff formula: `delay = basDelay * (2 ^ attemptIndex)` → `1000ms, 2000ms, 4000ms`
- `ErrorSnackbarListener` được mount tại `AppShell` (Phase 3.7) — phase này chỉ tạo provider + widget, chưa mount vào app
- Dependency của: tất cả feature datasource dùng Dio (Phase 3.2 YouTubeService, Phase 8.1 LyricsService...)
