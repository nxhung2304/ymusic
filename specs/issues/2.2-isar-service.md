## Description

Tạo `IsarService` — wrapper shared cho local database Isar.
Service này quản lý vòng đời database (open/close) và pattern đăng ký schema động theo từng feature.

Mục tiêu: các feature sau (Search cache, Lyrics cache, Downloads, History) chỉ cần đăng ký schema của mình vào `IsarService` mà không cần biết chi tiết khởi tạo DB.

- Singleton Isar instance được khởi tạo một lần tại app startup
- Schema được đăng ký dần theo feature (không import tất cả upfront)
- Cung cấp Riverpod provider để inject vào repository

## Notes

- Packages cần: `isar`, `isar_flutter_libs`, `path_provider` | dev: `isar_generator`
- Singleton pattern: dùng `static` field thay vì `get_it` hay global variable — đơn giản, không over-engineer
- Schema registration pattern: `main.dart` gom tất cả schemas → truyền vào `initialize`. Ví dụ khi Phase 3.4 thêm Song cache: `IsarService.initialize([SongSchema, ...])`
- Testing: dùng `isar_test` package hoặc tạo in-memory Isar instance — không dùng mock
- Đây là **peer** của `FirestoreService` (2.1) — đặt cùng `lib/core/services/`
- Dependency của: 3.4 (Song cache schema), 8.2 (Lyrics cache schema), 9.2 (DownloadedSong schema), 5.1 (History Isar schema)
- `isarServiceProvider` expose `Isar` instance trực tiếp để repository dùng — không cần expose `IsarService` class
