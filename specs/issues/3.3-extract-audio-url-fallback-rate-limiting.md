## Description

Thêm method `extractAudioUrl(String videoId)` vào `YouTubeDatasource` — dùng `youtube_explode_dart` để lấy audio stream URL từ một `videoId`.

Vì YouTube giới hạn request rate, method này phải có:
1. **Rate limiting**: throttle tối đa 1 request/giây thông qua một internal queue.
2. **Fallback**: nếu stream quality cao nhất thất bại, thử lần lượt các quality thấp hơn trước khi throw exception.

## Implementation Checklist

- [ ] Trong `lib/core/services/youtube_service.dart`:
  - Thêm internal `_RequestQueue` hoặc dùng `StreamController` / `Completer` để queue các request
  - Throttle: sau mỗi request, delay tối thiểu 1000ms trước khi xử lý request tiếp theo
  - Method `extractAudioUrl(String videoId) → Future<String>`:
    - Enqueue request, await turn
    - Gọi `YoutubeExplode.videos.streamsClient.getManifest(videoId)`
    - Lấy danh sách `AudioOnlyStreamInfo`, sort theo bitrate giảm dần
    - Thử từng entry: lấy `.url.toString()` — nếu URL hợp lệ (non-empty) thì return ngay
    - Nếu tất cả entry fail → throw `YouTubeException('No audio stream available for $videoId')`
  - Dispose `YoutubeExplode` instance đúng cách khi service bị dispose (nếu service có lifecycle)

- [ ] Tạo unit test tại `test/core/services/youtube_service_test.dart` (bổ sung vào file hiện có):
  - **Happy path**: mock `StreamManifest` trả về ≥1 `AudioOnlyStreamInfo` → assert URL trả về đúng
  - **Fallback**: mock stream đầu thất bại (empty URL hoặc throw), stream tiếp theo thành công → assert fallback URL được dùng
  - **All fail**: mock tất cả stream fail → assert `YouTubeException` được throw
  - **Rate limiting**: gọi 3 request đồng thời, assert thời gian hoàn thành ≥ 2000ms (chứng minh throttle 1 req/s đang hoạt động)
  - **Concurrency**: gọi nhiều request song song, assert không có race condition (results đúng thứ tự hoặc đúng giá trị)

## Notes

- Package: `youtube_explode_dart` — dùng `StreamManifest`, `AudioOnlyStreamInfo`
- Ưu tiên `AudioOnlyStreamInfo` (không có video) để giảm bandwidth; sort theo `bitrate.bitsPerSecond` giảm dần
- Rate limiting **không được drop request** — phải queue và xử lý tuần tự. Dùng `Completer<String>` trong queue hoặc một `StreamController` để pipeline các request.
- Giữ `YoutubeExplode` inject qua constructor (đã inject từ Phase 3.2) — không khởi tạo mới trong method
- Method này sẽ được dùng bởi:
  - Phase 3.5 (`AudioPlayerService`) — cần URL để feed vào `just_audio`
  - Phase 9.1 (`DownloadService`) — cần URL để download file
  - Phase 11.1 (`VideoPlayerScreen`) — extract video stream (tương tự nhưng khác method)
- Không cache URL ở đây — URL từ YouTube có TTL ngắn (~6h); cache sẽ không hiệu quả và gây lỗi khi URL hết hạn
- Đây là **`[👤]` task** — tập trung vào stream handling + rate limiting logic; không làm UI
