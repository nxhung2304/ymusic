## Description

Refactor `LoginScreen` to fix 4 critical architecture violations discovered in code review (score 5/10).
The screen currently calls datasource directly, catches exceptions in UI, and uses local `setState` instead of Riverpod state.

- Create `SignInWithGoogleUsecase` in `domain/usecases/`
- Create `AuthNotifier` with freezed `AuthState` (initial / loading / error / success)
- Move all exception → Failure mapping to `AuthRepositoryImpl`
- Refactor screen from `ConsumerStatefulWidget` → `ConsumerWidget` that watches `authNotifierProvider`
- Extract all 17 magic numbers to named `static const` file-level constants
