## Description

Tạo `playerStateProvider` — Riverpod provider quản lý toàn bộ player state: bài hát đang phát, trạng thái playback (playing/paused), vị trí hiện tại, queue (tạm thời hardcoded, Phase 5.7 sẽ thay bằng dynamic queue).

Provider này là **single source of truth** cho tất cả UI liên quan đến player (MiniPlayerBar, FullPlayerScreen). Dữ liệu đến từ `AudioPlayerService` (streams) + local state (current song, queue).

Sau khi implement, sẽ wire provider này vào UI components từ Phase 3.6 để hiển thị real-time playback state.

## Design

### State Structure

```dart
@freezed
class PlayerState with _$PlayerState {
  const factory PlayerState({
    required SongModel? currentSong,           // null nếu chưa phát gì
    required bool isPlaying,                   // true/false
    required Duration position,                // vị trí hiện tại
    required Duration duration,                // độ dài bài
    required List<SongModel> queue,            // danh sách bài tới
    required int queueIndex,                   // index bài hiện tại trong queue
  }) = _PlayerState;

  // Convenience getters
  bool get hasNext => queueIndex < queue.length - 1;
  bool get hasPrev => queueIndex > 0;
  SongModel? get nextSong => hasNext ? queue[queueIndex + 1] : null;
}
```

### Provider Architecture

```
playerStateProvider (riverpod notifier)
  ├─ Reads: audioPlayerServiceProvider (streams)
  ├─ Reads: currentSongProvider (watch)
  └─ Exposes: PlayerState (reactive)
```

**Lý do Notifier:** Cần trạng thái mutable (currentSong, queue index, position) + methods để update.

### 2. State Class — `lib/features/player/domain/models/player_state.dart`

```dart
import 'package:freezed_annotation/freezed_annotation.dart';
import '../../domain/models/song_model.dart';

part 'player_state.freezed.dart';

@freezed
class PlayerState with _$PlayerState {
  const factory PlayerState({
    required SongModel? currentSong,
    required bool isPlaying,
    required Duration position,
    required Duration duration,
    required List<SongModel> queue,
    required int queueIndex,
  }) = _PlayerState;

  const PlayerState._();

  bool get hasNext => queueIndex < queue.length - 1;
  bool get hasPrev => queueIndex > 0;
  SongModel? get nextSong => hasNext ? queue[queueIndex + 1] : null;

  double get progress {
    if (duration.inMilliseconds == 0) return 0.0;
    return (position.inMilliseconds / duration.inMilliseconds).clamp(0.0, 1.0);
  }
}
```

### 3. Notifier — `lib/features/player/presentation/providers/player_notifier.dart`

```dart
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:riverpod_annotation/riverpod_annotation.dart';
import '../../domain/models/player_state.dart';
import '../../domain/models/song_model.dart';
import '../../../../core/services/audio_player_service.dart';

part 'player_notifier.g.dart';

@riverpod
class PlayerNotifier extends _$PlayerNotifier {
  @override
  PlayerState build() {
    _setupStreamListeners();
    return PlayerState(
      currentSong: null,
      isPlaying: false,
      position: Duration.zero,
      duration: Duration.zero,
      queue: [],
      queueIndex: -1,
    );
  }

  void _setupStreamListeners() {
    final audioService = ref.watch(audioPlayerServiceProvider);

    // Listen to playback state
    ref.listen(
      audioPlayerServiceProvider.select((service) => service.playerStateStream),
      (previous, next) {
        next.listen((playerState) {
          state = state.copyWith(isPlaying: playerState.playing);
        });
      },
    );

    // Listen to position updates
    ref.listen(
      audioPlayerServiceProvider.select((service) => service.positionStream),
      (previous, next) {
        next.listen((position) {
          state = state.copyWith(position: position);
        });
      },
    );

    // Listen to duration updates
    ref.listen(
      audioPlayerServiceProvider.select((service) => service.durationStream),
      (previous, next) {
        next.listen((duration) {
          state = state.copyWith(duration: duration ?? Duration.zero);
        });
      },
    );
  }

  void setCurrentSong(SongModel song) {
    state = state.copyWith(
      currentSong: song,
      position: Duration.zero,
      duration: Duration.zero,
      queueIndex: 0,
    );
  }

  void addToQueue(SongModel song) {
    final updatedQueue = [...state.queue, song];
    state = state.copyWith(queue: updatedQueue);
  }

  void clearQueue() {
    state = state.copyWith(
      queue: [],
      queueIndex: -1,
    );
  }

  void clearCurrentSong() {
    state = state.copyWith(currentSong: null);
  }

  void updatePosition(Duration position) {
    state = state.copyWith(position: position);
  }
}
```

import 'package:flutter_test/flutter_test.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:mocktail/mocktail.dart';
// ... imports

void main() {
  group('PlayerNotifier', () {
    late MockAudioPlayerService mockAudioService;
    late ProviderContainer container;

    setUp(() {
      mockAudioService = MockAudioPlayerService();
      // Setup mock streams
      when(() => mockAudioService.playerStateStream).thenAnswer(
        (_) => Stream.value(PlayerState(...)),
      );
      when(() => mockAudioService.positionStream).thenAnswer(
        (_) => Stream.value(Duration.zero),
      );
      when(() => mockAudioService.durationStream).thenAnswer(
        (_) => Stream.value(Duration(minutes: 3)),

        overrides: [
          audioPlayerServiceProvider.overrideWithValue(mockAudioService),
        ],
      );
    });

    test('initial state is correct', () {
      final state = container.read(playerNotifierProvider);
      expect(state.currentSong, null);
      expect(state.isPlaying, false);
      expect(state.queue, isEmpty);
    });

    test('setCurrentSong updates state', () {
      final song = SongModel(...);
      container.read(playerNotifierProvider.notifier).setCurrentSong(song);

      final state = container.read(playerNotifierProvider);
      expect(state.currentSong, song);
      expect(state.queueIndex, 0);
    });

    test('addToQueue appends song', () {
      final song1 = SongModel(...);
      final song2 = SongModel(...);

      container.read(playerNotifierProvider.notifier).addToQueue(song1);
      container.read(playerNotifierProvider.notifier).addToQueue(song2);

      final state = container.read(playerNotifierProvider);
      expect(state.queue.length, 2);
      expect(state.queue[0], song1);
      expect(state.queue[1], song2);
    });

    test('clearQueue resets queue and index', () {
      // Setup with items
      container.read(playerNotifierProvider.notifier).addToQueue(SongModel(...));

      container.read(playerNotifierProvider.notifier).clearQueue();

      final state = container.read(playerNotifierProvider);

      expect(state.queueIndex, -1);

      final song = SongModel(...);

      container.read(playerNotifierProvider.notifier).setCurrentSong(song);

      final state = container.read(playerNotifierProvider);
      expect(state.hasNext, true);
    });

    test('hasPrev is true when queueIndex > 0', () {
      // Setup with multiple items
      final song = SongModel(...);
      container.read(playerNotifierProvider.notifier).addToQueue(song);

      container.read(playerNotifierProvider.notifier).setCurrentSong(song);

// MiniPlayerBar example
class MiniPlayerBar extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final playerState = ref.watch(playerStateProvider);
    final currentSong = playerState.currentSong;

    if (currentSong == null) return SizedBox.shrink();

    return Container(
      // UI
      child: Row(
        children: [
          Image.network(currentSong.thumbnailUrl),
          Column(
            children: [
              Text(currentSong.title),
              Text('${playerState.position.inSeconds}s'),
            ],
          ),
          IconButton(
            onPressed: () => ref.read(playerNotifierProvider.notifier)
                .setCurrentSong(currentSong),
            icon: Icon(playerState.isPlaying ? Icons.pause : Icons.play_arrow),
          ),
        ],
      ),
    );
  }
}

## Notes

- **Phase 3.8 depends on:** Phase 3.7 (`AudioPlayerService`), Phase 3.1 (`SongModel`)
- **Phase 3.8 is used by:** Phase 3.6 (wire UI), Phase 3.9 (wire SearchScreen), Phase 5.7 (dynamic queue), Phase 3.11 (unit test)
- **Stream subscription:** Dùng `ref.listen()` để reactive update state từ `AudioPlayerService` streams
- **Queue là state local:** Phase 3 tạm hardcode empty queue, Phase 5.7 sẽ implement `QueueProvider` dynamic
- **Why Notifier?** Cần state mutable (setCurrentSong, addToQueue) + computed properties (hasNext, progress)
- **Why keepAlive?** Player state phải persist suốt app lifecycle, không bị reset khi navigate away
- **Breaking point:** Phase 3.6 (Player UI) cần provider này để display real-time state
- **Freezed:** Dùng `copyWith()` để immutable updates
- **Convenience providers:** `currentSongProvider`, `isPlayingProvider`, etc. cho UI chỉ cần 1 field mà không rebuild khi field khác change
