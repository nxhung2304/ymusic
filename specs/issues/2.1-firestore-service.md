## Description

Tạo `FirestoreService` — một wrapper shared cho Firestore CRUD operations.
Service này là nền tảng cho tất cả các feature slice sau (LikedSongs, History, Playlists, Podcasts, Preferences).

Mục tiêu: tránh repeat boilerplate Firestore code trong từng repository, chuẩn hóa cách đọc/ghi/xóa document.

## Acceptance Criteria
- [ ] `setDocument(path, data)` — ghi/overwrite một document tại đường dẫn bất kỳ
- [ ] `getDocument(path)` — đọc document, trả về `Map<String, dynamic>?` (null nếu không tồn tại)
- [ ] `deleteDocument(path)` — xóa document tại đường dẫn
- [ ] `getCollection(path)` — đọc toàn bộ documents trong collection, trả về `List<Map<String, dynamic>>`
- [ ] `streamDocument(path)` — expose `Stream<Map<String, dynamic>?>` để reactive listen
- [ ] `streamCollection(path)` — expose `Stream<List<Map<String, dynamic>>>` để reactive listen collection
- [ ] Tất cả methods đều wrap try-catch và throw `FirestoreException` rõ ràng khi fail
- [ ] `flutter analyze` — 0 warnings

## Notes

- Package cần: `cloud_firestore`
- Path convention theo Firestore schema: `/users/{uid}/liked_songs/{songId}`, `/users/{uid}/playlists/{playlistId}/songs/{songId}`, ...
- `setDocument` nên support `merge: true` để partial update (không overwrite toàn bộ document)
- `streamCollection` dùng cho real-time UI (LibraryScreen tabs)
- Đây là **shared service** — đặt trong `lib/core/services/`, không phải trong feature folder
- Dependency của: 2.2 (IsarService peer), Phase 6 (LikedSongsRepository), Phase 7 (PlaylistRepository), Phase 10 (UserPreferenceRepository)
